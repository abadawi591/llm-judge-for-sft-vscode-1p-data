## Engine Messages Notes

The `GitHub.copilot.chat/engine.messages` event snapshots the current LLM API call context and conversation state. A few important nuances:

- The `messagesJson` array contains the full conversation so far; however, only the last item corresponds to the current API call.
- Model fields are mutually exclusive in practice:
  - `baseModel` appears for assistant completions (response model)
  - `request.option.model` appears for requests (user/tool/system phases); sometimes serialized as a quoted JSON string
- The `messageSource` field explains why the call happened. Observed values include VS Code–initiated flows, such as `intentDetection`, which represent background calls triggered by the editor rather than a direct user prompt. These calls may appear alongside user-initiated ones in the same session.
- When reconstructing per-message models, attach the model only to the last message of the snapshot (the current request/response), not to historical messages.

### Identifier Correlation
- Per-turn correlation uses request identifiers:
  - `GitHub.copilot-chat/conversation.messageText.headerRequestId` equals `GitHub.copilot.chat/engine.messages.headerRequestId` for the same turn.
  - `GitHub.copilot-chat/interactiveSessionMessage.requestId` equals `GitHub.copilot.chat/engine.messages.headerRequestId`.
  - `GitHub.copilot-chat/interactiveSessionMessage.sessionId` equals `GitHub.copilot.chat/engine.messages.conversationId`.
- These mappings enable reliable joins without heuristics (e.g., to tag per-message `mode` from `conversation.messageText` onto the last user message of the corresponding `engine.messages` turn).
- Caveat: telemetry for a single turn can be split across different files. In-file checks may show many “unmatched” pairs despite the identifiers being equivalent. Cross-file correlation closes these gaps.
  - Verification: `python cli.py check header-request-id-match <telemetry_dir> --sample 2000 --seed 42 --regex "^[^.].*\.json$"`

### Per‑snapshot Model Attribution (Do this, don’t backfill older turns)
- Always attribute model to the last message in each `engine.messages` snapshot:
  - If last role is `assistant`: model = `baseModel` (fallback to unwrapped `request.option.model`).
  - If last role is `user`/`tool`/`system`: model = unwrapped `request.option.model` (requested model).
- For precise joins with interactive session telemetry:
  - Group by conversation: `interactiveSession.sessionId == engine.conversationId`.
  - Within that conversation, map to a specific `engine.messages` by exact `requestId`.
