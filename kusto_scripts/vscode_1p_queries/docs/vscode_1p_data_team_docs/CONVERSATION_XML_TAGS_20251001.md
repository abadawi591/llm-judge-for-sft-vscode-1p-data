## Key Takeaways

1. Always extract actual input from `<userRequest>`
2. Many user messages contain system-provided context outside `<userRequest>`
3. Tool responses use distinct tags like `<execution-summaries>` and `<cell-output>`
4. Assistant replies are largely plain text

## Appendix

### User Input Isolation

```xml
<!-- In a user message: -->
<context>System-provided context</context>
<reminderInstructions>System instructions</reminderInstructions>
<userRequest>
  This is what the user actually typed
</userRequest>
```

- Everything inside `<userRequest>` is user-authored.
- Everything outside `<userRequest>` is system-provided context.

### Attachments Example

```xml
<attachments>
  <attachment filePath="c:\\Users\\user\\file.ipynb"></attachment>
</attachments>
```

- `attachment.filePath`: absolute path of the attached file.
- `attachment.index` (optional): file index when multiple attachments are present.

### Conversation Continuation Example

```xml
<conversation-summary>
  <analysis>Previous conversation analysis</analysis>
  <summary>Brief summary of the conversation</summary>
</conversation-summary>
<importantReminders>Critical points</importantReminders>
<requirementsUnderstanding>Requirements to remember</requirementsUnderstanding>
<responseModeHints>Preferred response style</responseModeHints>
<userRequest>New user input…</userRequest>
```
